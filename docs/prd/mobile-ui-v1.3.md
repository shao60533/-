# PRD: 移动端信息架构与 UI 实装 v1.3

| 项 | 值 |
|---|---|
| Feature | `mobile-ui-v1.3` |
| 版本 | v1.3 |
| 日期 | 2026-05-09 |
| Demo 契约 | [../../demo_mobile_full_v1.html](../../demo_mobile_full_v1.html) |
| 范围 | 移动端 IA / UI / 用户动线重组 |
| 硬约束 | 不改产品功能、不改后端能力、不改技术架构 |

## 1. 背景

当前移动端页面已经具备功能覆盖，但信息组织仍偏“桌面功能平铺”：首页和持仓重复、纸面交易入口过深、分析详情里核心结论与原文混杂、选股结果默认展开过多专家细节，导致用户在手机上需要大量滚动和判断。

本 PRD 以 `demo_mobile_full_v1.html` 当前形态为唯一视觉和信息架构准绳，将现有功能重新组织为移动端可执行流程。该版本不新增任何后端接口、任务类型、数据模型或投资逻辑。

## 2. 目标

1. 首页承载账户概览 + 持仓明细，取消“首页/持仓”重复入口。
2. 纸面交易升为底部一级入口，符合“分析 -> 纸面追踪”的核心动线。
3. 分析详情以 8 个结构化分析卡为核心，不把原文混入结构化 tab。
4. 选股结果默认高密度概览，大师评分默认折叠。
5. More 只承载低频页面入口，不展示运营状态卡和调度器快捷卡。

### 2.1 成功指标

| 指标 | 目标 |
|---|---|
| 底部一级入口 | 固定为：首页 / 分析 / 发现 / 纸面 / 更多 |
| 首页默认持仓展示 | 默认 5 张，可切换全部 9 张 |
| Analysis 结构化 tabs | 严格 8 个：概览 / 市场 / 情绪 / 新闻 / 基本面 / 辩论 / 风险 / 决策 |
| Screener 大师详情 | 默认收起，summary 暴露共识摘要 |
| 横向溢出 | 375 / 390 / 430 / 768px 视口均为 0 |
| 功能变更 | 0 个新增后端能力，0 个新增任务类型，0 个新增数据库字段 |

## 3. 范围

### 3.1 In Scope

**移动端导航**

- 底部 tab 改为：首页、分析、发现、纸面、更多。
- 独立“持仓”不再作为移动端一级 tab。
- 纸面交易列表页作为一级 tab 页面，不显示“返回更多”。
- More 页面只保留低频入口：报告中心、策略回测、交易记录、预警中心、任务中心、系统设置、账号。

**首页**

- 首页顶部保留账户总览：账户总值、今日 PnL、总盈亏、收益率、活跃预警、净值曲线。
- 首页合并持仓管理：
  - 搜索股票、买入入口。
  - `显示 5 / 全部 9` 切换。
  - 交易记录入口。
  - 纸面计划入口。
  - 批量分析持仓入口以“产品缺口”标注，不伪装为已实装能力。
  - 持仓卡展示成本、现价、盈亏、市值、仓位/状态。
  - 持仓卡提供：看分析、卖出、修正成本、移除。
- 首页保留净值与仓位、快捷操作。
- 首页不展示“跨页面待办聚合 / 今日决策队列”。

**分析**

- 分析首页保留发起分析表单、标准/深度切换、分析记录 Inbox。
- 删除独立 `Inbox 工具` 区块：刷新、筛选、看任务。
- 分析详情首屏直接展示结构化分析核心。
- 结构化核心只包含 8 tabs，不包含“原文”。
- 原始报告保留在 `原始报告 fallback / debug` 区，含 Markdown / 市场原文 / 新闻原文 / 风险原文。
- Quick Info 区块不展示。
- 顶部重复结论大卡不展示；记录与操作下移到 K 线后。

**发现 / 智能选股 V3**

- 发现页保留自然语言描述、市场切换、高级配置、大师列表、模式、成本估算。
- 表单底部只保留主按钮 `开始筛选`，不展示取消按钮。
- 最近选股保留 success-only 历史和 running 行缺口标注。
- 选股结果页不展示透明度链路 / 审计信息。
- 候选排名默认展示候选概览。
- 14 位大师评分默认折叠，summary 文案为：
  `查看 14 位大师评分（共识 64% / 看多 9 / 看空 2 / 中性 3）`。

**纸面交易**

- 纸面交易列表页作为一级页面，不展示“返回更多”。
- 纸面交易详情保留“返回纸面交易”。
- 详情页不展示 `策略 / 日度数据` 页内 tab。
- 日度数据区只保留 `刷新日度数据` 一个按钮。
- AI 决策内容使用分析核心同风格结构化 UI，不展示英文 raw text。
- Plan / Event 双视图保留在执行记录中。

**更多**

- More 页不展示“复盘与运营”状态卡。
- More 页不展示“调度器”快捷入口。
- 系统设置入口副标题使用“模型与通知”。

**覆盖矩阵**

- Coverage 仍作为 demo 审计页存在，但不是用户主入口。
- Coverage 必须诚实标注真实产品缺口，不把 demo 建议写成已实装功能。

### 3.2 Out of Scope

| 不做 | 原因 |
|---|---|
| 新增后端 API | 本 PRD 只做移动端信息架构与 UI 实装 |
| 新增数据库字段 | 无数据模型变更 |

| 实装 Diagnostics / Schwab OAuth 面板 | 当前真实产品缺口，保留在 Coverage gap |
| 新增 Ticker 聚合页 | 非既有能力，不进入本 PRD |

## 4. 信息架构

### 4.1 底部导航

| Tab | 对应内容 | 说明 |
|---|---|---|
| 首页 | 账户 + 持仓 | Dashboard 与 Portfolio 能力合并展示 |
| 分析 | 发起分析 + Inbox + 详情 | 结构化报告为核心 |
| 发现 | Screener V3 | 自然语言选股 + 大师 Agent |
| 纸面 | Paper trade list/detail | 前向追踪和历史回放升级为一级入口 |
| 更多 | 低频页面 | 报告、回测、交易记录、预警、任务、设置、账号 |

### 4.2 页面主次关系

| 页面 | 首屏优先级 | 默认折叠/下移 |
|---|---|---|
| 首页 | 资产总览、持仓前 5、产品缺口提示 | 全部持仓、净值细节、快捷操作 |
| 分析详情 | 结构化核心 8 card、K 线 | 元信息、原始报告 |
| 发现结果 | 候选排名、综合分、投票条 | 14 大师详情、圆桌摘要 |
| 纸面详情 | 当前策略、日度数据、结构化决策 | Plan/Event 执行明细 |
| 更多 | 页面入口 | 状态卡、调度器快捷入口不展示 |

## 5. 用户故事与验收

### US-MUI-1：首页合并持仓

> 作为移动端用户，我希望打开首页就能看到资产状态和关键持仓明细，而不是在首页和持仓两个 tab 之间来回切。

**验收**：

- 底部 tab 没有“持仓”。
- 首页持仓区默认显示 5 张卡：NVDA、TSLA、AAPL、MSFT、AVGO。
- 点击 `全部 9` 后显示 LLY、META、AMD、TSM。
- 点击 `显示 5` 后恢复默认 5 张。
- 每张持仓卡都有 `看分析` 入口。
- NVDA / TSLA 至少保留卖出、修正成本、移除操作。

### US-MUI-2：纸面交易成为一级入口

> 作为移动端用户，我希望从底部 tab 直接进入纸面交易，快速查看追踪计划和回放。

**验收**：

- 底部第 4 个 tab 为 `纸面`。
- 纸面列表页顶部不展示 `返回更多`。
- 纸面详情仍可 `返回纸面交易`。
- 纸面详情顶部不展示 `策略 / 日度数据` 两个页内 tab。

### US-MUI-3：分析详情结构化优先

> 作为移动端用户，我希望进入分析详情后先看到结构化结论，而不是重复摘要或原始文本。

**验收**：

- AnalysisDetail 顶部第一个业务区为 `结构化分析核心`。
- tabs 严格为 8 个：概览、市场、情绪、新闻、基本面、辩论、风险、决策。
- 不存在结构化 tab `原文`。
- Quick Info 不展示。
- 记录与操作在 K 线后展示。
- 原始报告只出现在 fallback/debug 区。

### US-MUI-4：选股结果默认轻量

> 作为移动端用户，我希望先看到候选股票排名和共识摘要，需要时再展开 14 位大师详情。

**验收**：

- 选股结果不展示透明度链路 / 审计信息。
- 14 位大师评分详情默认收起。
- details summary 包含共识摘要：共识 64% / 看多 9 / 看空 2 / 中性 3。
- 展开后仍能看到 14 位大师逐项结论。

### US-MUI-5：More 只做低频入口

> 作为移动端用户，我希望 More 是清晰的功能地图，而不是混入运营状态或调度器信息。

**验收**：

- More 页不展示“复盘与运营”。
- More 页不展示“任务失败 / 报告生成”状态卡。
- More 页不展示“调度器”快捷入口。
- 系统设置入口副标题为“模型与通知”。

## 8. 实装约束

1. 不新增后端 endpoint。
2. 不新增数据库 migration。
3. 不新增 task type。
4. 不改变已有 API response schema。
5. 不改变分析、选股、纸面交易、回测、报告的业务逻辑。
6. 不改变 LLM provider / preset 行为。
7. 不引入新的前端框架或架构形态。
8. 允许新增/整理前端展示组件，但必须复用现有 React island、shadcn 风格组件和 CSS token。
9. 移动端不暴露独立持仓一级入口；生产兼容路由/API 可保留。
10. demo 标注为缺口的能力，不得在实装时伪装成已完成。

## 9. 验收清单

### 9.1 功能不变

- 现有分析提交、运行中、详情查看可用。
- 现有 Screener V3 表单、运行态、结果查看可用。
- 现有 Paper trade list/detail 可用。
- 现有交易记录、报告、回测、预警、任务、设置入口可用。
- 现有模型切换两段式 UI 可用。

### 9.2 UI 对齐 demo

- 底部导航文案与顺序一致。
- 首页持仓默认 5 张，与 demo 同类信息密度。
- AnalysisDetail 无 Quick Info，无结构化“原文”tab。
- Screener result 无透明度审计块，大师评分默认收起。
- Paper list 无返回更多，Paper detail 无策略/日度 tabs。
- More 无复盘运营卡、无调度器入口。

### 9.3 移动端质量

- 375px、390px、430px、768px 下 `document.body.scrollWidth <= viewportWidth`。
- 所有主按钮高度 >= 44px。
- 横滑 chips 不撑破页面。
- 文案不遮挡、不截断关键数字。
- 底部 tab 不与内容重叠。

## 10. 风险与处理

| 风险 | 影响 | 处理 |
|---|---|---|
| Dashboard 合并持仓导致首页过长 | 用户滚动成本上升 | 默认只显示 5 张，全部 9 需主动展开 |
| 移除持仓一级入口被误解为删功能 | 用户找不到持仓管理 | 首页明确命名“持仓明细”，More 保留交易记录入口 |
| 纸面交易升一级后 More 入口减少 | 低频用户认知变化 | 底部 tab 文案直接使用“纸面” |
| 结构化报告缺 rendering 数据 | 老分析详情可能为空 | fallback/debug 原始报告保留 |
| 持仓看分析无专用 ticker 聚合页 | 跳转体验不完整 | 复用现有 Analysis 入口/详情，不新增 ticker page |
