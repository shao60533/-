# 股票辅助决策系统 — UI/UX 重设计方案

## 一、现状诊断

### 1.1 结构性问题

经过对 `index.html`（210行）、`style.css`（700+行）、`app.js`（192行）、`app.py`（57行/损坏状态）的完整审读，发现以下核心问题：

| 维度 | 现状 | 严重程度 |
|------|------|----------|
| 页面完整度 | 9个页面中仅 Dashboard 有实际 HTML，其余 8 个页面为空壳 | 致命 |
| 前端逻辑 | app.js 90% 函数为 `console.log` 占位符，无 API 调用 | 致命 |
| 后端 API | 仅 dashboard/portfolio/price 三组端点，缺少分析/选股/预警/报告/回测/设置 | 严重 |
| 数据可视化 | ECharts 已引入但未初始化任何图表 | 严重 |
| 状态管理 | 无 loading/error/empty 状态处理 | 中等 |
| 前后端协作 | Socket.IO 已连接但无实际推送事件 | 中等 |

### 1.2 值得保留的部分

- 暗色交易主题（CSS 变量体系完整，配色专业）
- 侧边栏 + 移动端底部 Tab Bar 的双端导航架构
- 移动端 "更多" 底部弹出面板（sheet）交互模式
- 全局搜索栏的 UI 框架

---

## 二、设计原则

1. **信息密度优先** — 交易系统用户需要在一屏内看到尽可能多的关键数据，而不是大留白
2. **操作路径最短** — 从"看到信号"到"执行动作"不超过 2 次点击
3. **实时感** — 价格、盈亏、预警状态必须有实时更新的视觉反馈（脉冲动画、渐变色变）
4. **渐进披露** — 概览层展示结论（BUY/SELL/HOLD），详情层展示推理过程（AI 多 Agent 报告）
5. **移动端不是缩小版** — 移动端重新编排信息层级，用卡片列表替代表格

│                          │                      │
├──────────────────────────┴──────────────────────┤

**改进点**：

- 持仓表格：接 `/api/portfolio/holdings` + 实时价格推送，盈亏颜色动态变化

## 六、移动端适配策略（重点）

> **设计前提**：本系统将部署到云端，主要通过手机操作。移动端不是桌面的"缩小版"，而是**首要交互场景**。所有页面的设计、测试、优化均以移动端优先（Mobile-First）为原则。

### 6.1 断点与布局策略

| 断点 | 范围 | 布局 | 目标设备 |
|------|------|------|----------|
| `xs` | < 375px | 单列极简 | 小屏手机（SE） |
| `sm` | 375px ~ 767px | 单列标准 | 主流手机（iPhone/Android） |
| `md` | 768px ~ 1199px | 双列混合 | 平板横屏 |
| `lg` | ≥ 1200px | 侧边栏 + 多列 | 桌面 |

**核心原则**：CSS 从 `sm`（手机）写起，通过 `@media (min-width: 768px)` 向上扩展桌面样式。

### 6.2 导航系统

**底部 Tab Bar**（移动端核心导航）：

```
┌───────────────────────────────────────────────┐
│  仪表盘    分析     选股     持仓     更多       │
│   🏠      🧠      🔍      💼      ···       │
└───────────────────────────────────────────────┘
   固定底部 · 高度 56px · safe-area-inset-bottom
```

| 属性 | 值 |
|------|-----|
| 高度 | 56px + `env(safe-area-inset-bottom)` 适配 iPhone 刘海/底部横条 |
| 背景 | `rgba(13,17,23,0.95)` + `backdrop-filter: blur(12px)` 磨砂半透明 |
| 边框 | 顶部 1px `rgba(48,54,61,0.6)` |
| Tab 项 | 图标 20px + 文字 10px，垂直居中，激活态 `var(--accent-blue)` + 发光 |
| 点击区 | 最小 44px × 44px（WCAG 触摸目标标准） |
| "更多"弹窗 | 底部上滑 sheet，高度 auto，圆角顶部 16px，内含网格排列的次级功能入口 |

**"更多" Sheet 内网格**：
